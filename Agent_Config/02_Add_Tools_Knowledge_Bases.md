# Agent Foundry - Add Knowledge Base Tools Configuration

## Campaign Strategist & Planner Agent - Knowledge Base Tools

This guide provides the configuration for all 12 knowledge base tools that give your agent access to comprehensive marketing expertise.

---

## üìã Overview

Knowledge Base tools allow your agent to query and access information from uploaded knowledge bases. Each tool connects to a specific knowledge base file.

**Total Tools:** 12
**Tool Type:** Knowledge Base
**Target Function:** List columns (for all tools)

---

## üîß Tool Configuration Instructions

For each tool below:

1. Navigate to **Tools** section in Agent Foundry
2. Click **Add Tool**
3. Fill in the fields exactly as specified
4. **Target** should always be: **Knowledgebase**
5. **Target function** should always be: **List columns**
6. Save the tool configuration

## üéØ Tool Naming Conventions

### Function Name Pattern

All function names follow this pattern:
```
kb_[topic]
```

**Examples:**
- `kb_campaign_fundamentals`
- `kb_meta_advertising`
- `kb_audience_targeting`

**Benefits:**
- Consistent naming
- Easy to identify KB tools
- Clear topic indication
- Prevents naming conflicts

## üí° Pro Tips

### Tool Descriptions

**Write effective tool descriptions:**
- ‚úÖ Be specific about what data is available
- ‚úÖ Mention key topics and frameworks
- ‚úÖ Include search keywords the agent might use
- ‚úÖ Keep under 200 characters for best performance

**Example:**
```
Good: "Access Meta advertising best practices including campaign
structure, objectives, targeting, ad formats, and optimization"

Bad: "Meta information"
```

### Knowledge Base Organization

**Best practices:**
1. Keep related information together
2. Split large topics (e.g., Creative Part 1 & 2)
3. Stay under 18,000 character limit
4. Use clear, descriptive KB names
5. Maintain consistent formatting

### Testing Tools

**Test each tool:**
```
Test Prompt: "What are the best practices for Meta advertising?"
Expected: Agent uses kb_meta_advertising tool
Verify: Response includes Meta-specific information

Test Prompt: "How should I measure campaign success?"
Expected: Agent uses kb_metrics_kpis tool
Verify: Response includes KPI frameworks and benchmarks
```

## ‚ö†Ô∏è Common Issues

### Issue: Agent doesn't use knowledge base tools

**Solutions:**
- Check tool names match KB names exactly
- Verify KBs are uploaded and accessible
- Ensure system prompt mentions knowledge bases
- Test with more specific prompts

### Issue: KB tool returns no data

**Solutions:**
- Verify KB file uploaded successfully
- Check KB file size (under 18,000 chars)
- Ensure KB has proper Markdown formatting
- Test KB file with validation tool: `npm run validate:kb`

### Issue: Wrong tool selected by agent

**Solutions:**
- Make tool descriptions more distinct
- Add specific keywords to descriptions
- Reduce overlap between tool descriptions
- Test with clearer, more specific prompts
